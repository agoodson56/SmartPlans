## 3. Cache Expiration Fix (FIX #21)

### The Problem
When the Gemini context cache expired mid-analysis (TTL timeout during long bids), every subsequent API call got 403 "CachedContent not found". The system treated this as a rate limit:
1. Exhausted slot → tried next key → same 403
2. Burned through all 5 retry slots
3. Circuit breaker tripped (60s pause)
4. After 60s, tried again → same 403 → repeat forever
5. Even the flash fallback got 403 because it also referenced the dead cache

This caused 20+ minutes of spinning with 100+ wasted API calls on the Martinez bid.

### The Fix (3 changes in `ai-engine.js`)

**Change 1 — Detection** (line ~2120):
```javascript
if (errStatus === 403 && chunk._debug && chunk._debug.includes('CachedContent not found')) {
    throw { _cacheExpired: true, status: 403, message: 'Context cache expired — invalidating' };
}
```
Catches the specific 403 BEFORE it hits the generic retryable handler.

**Change 2 — Rethrow** (line ~2163):
```javascript
if (e._cacheExpired) throw e;
```
Ensures cache expiration errors bubble up past the SSE chunk parser.

**Change 3 — Global Invalidation** (line ~2219):
```javascript
if (err._cacheExpired) {
    if (this._contextCache) {
        console.warn(`[Brain:${brainDef.name}] ⚠️ Context cache EXPIRED — invalidating cache for all remaining brains`);
        this._contextCache = null;
    }
    break; // → fallback builds fresh body without _cacheName
}
```
Kills the cache reference globally. All subsequent brains and page scans send files directly instead of referencing the dead cache. Falls back to model fallback immediately (no wasted retries).

### Result
What was 20+ minutes of infinite loop now resolves in ONE API call: detect → invalidate → fallback → continue.

## 4. Transit Bid Price Fix (FormulaEngine3D → Proposal)

### The Problem
Two different numbers were showing:
- **Master Report / Proposal:** $1,794,281 (from `_computeFullBreakdown` with generic markups)
- **3D Formula Engine Card:** $1,880,665 (from FormulaEngine3D calibrated to actual Amtrak wins)

FormulaEngine3D was intentionally disabled as a pricing source because for non-transit projects, it double-marks AI BOM prices (which are already near-sell). But for transit projects, FormulaEngine3D is calibrated against actual winning Amtrak bids and IS the correct number.

### The Fix

**File:** `export-engine.js` (lines ~526-541)
- When `state.isTransitRailroad` is true and FormulaEngine3D produces a result > $1,000, it returns that as the bid price (early return before `_computeFullBreakdown`)
- Non-transit projects: unchanged, FormulaEngine3D remains reference-only

**File:** `proposal-generator.js` (lines ~1068-1073)
- Added matching Priority 2 check: if transit and `_engine3DResult.grandTotalSELL > 1000`, use it
- Previous Priority 2 (deterministic BOM) becomes Priority 3

### Priority Order (Transit)
1. **Bid Strategy** — if user explicitly applied per-category markups (unchanged)
2. **FormulaEngine3D** — calibrated to Emeryville/Martinez/Sacramento actuals (NEW)
3. **Deterministic BOM** — fallback if FormulaEngine fails
4. **Financial Engine AI** — last resort
5. **Raw BOM + 10%** — emergency fallback

### Priority Order (Non-Transit) — UNCHANGED
1. Bid Strategy
2. Deterministic BOM computation
3. Financial Engine AI
4. Raw BOM + 10%

### Circuit Breaker Trips
**Impact:** Was cascading (each trip added 60s delay), now fixed. The 403s were being counted as 429s (rate limits), causing the circuit breaker to trip repeatedly. With FIX #21, cache expiration is handled before it reaches the circuit breaker.

## Architecture Reference

### How Transit Pricing Flows
```
company-profile.js          → Calibration data (25 bids, equipment costs, benchmarks)
        ↓
formula-engine-3d.js        → SAGE markups + transit adders + per diem/mileage + benchmark calibration
        ↓
export-engine.js            → For transit: returns FormulaEngine3D total as bid price
        ↓
app.js (generateMasterReport) → Shows Total Bid in header
        ↓
proposal-generator.js       → Same total in proposal document
```

### How Cache Expiration Now Works
```
API returns 403 "CachedContent not found"
    ↓
ai-engine.js detects _cacheExpired (line ~2120)
    ↓
Catch block: this._contextCache = null (global kill)
    ↓
Break to model fallback → fresh request without _cacheName
    ↓
All subsequent brains see _contextCache === null → send files directly
```

### Key Variables
- `state.isTransitRailroad` — Toggle for transit/railroad pricing mode
- `state._engine3DResult` — FormulaEngine3D output (stored by export-engine)
- `this._contextCache` — Gemini context cache reference (null after expiration)
- `_currentUser?.is_admin` — Gates the analysis timer visibility
- `PRICING_DB.amtrakBenchmarks.actualBids` — Winning bid data for calibration
